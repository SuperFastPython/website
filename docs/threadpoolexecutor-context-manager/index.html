How to Use the ThreadPoolExecutor Context Manager

November 19, 2021 Python ThreadPoolExecutor

You can shutdown the ThreadPoolExecutor automatically by using a context manager.

In this tutorial, you will discover how to use context managers with thread pools in Python.

Let's get started.

Need to Automatically Shut Down the ThreadPoolExecutor

The ThreadPoolExecutor in Python provides a pool of reusable threads for executing ad hoc tasks.

You can submit tasks to the thread pool by calling the submit() function and passing in the name of the function you wish to execute on another thread. You can also submit tasks by calling the map() function and specifying the name of the function to execute and the iterable of items to which your function will be applied.

Once all of your tasks are finished, you must close the ThreadPoolExecutor in order to release all of the resources held by the worker threads, such as the memory used by each thread.

You can shut down a ThreadPoolExecutor by calling the shutdown() function directly.

For example:

...
# shutdown a thread pool
executor.shutdown()

If you forget to shut down the thread pool, the worker threads will remain active, waiting for work, and their resources will not be released, perhaps until your program closes.

This raises the question of whether we can automatically close the thread pool when we are finished using it.

How can we automatically shut down the ThreadPoolExecutor?

How To Use the ThreadPoolExecutor Context Manager

A ThreadPoolExecutor can be shut down automatically by using the context manager.

Objects in Python can implement a context manager, which lets you use the with statement to define a block in which you intend to use the object, after which a clean-up function will be called automatically once the block is exited.

You can learn more about Python context managers here:

  • contextlib — Utilities for with-statement contexts

This is a common pattern when opening a socket connection or a local file; for example:

...
# example of a generic context
with open('path/to/file.txt') as file:
	# do things
# file is closed automatically

The ThreadPoolExecutor can be used as a context manager.

When the block of the context manager is exited, the shutdown() function on the ThreadPoolExecutor will be called with the default arguments. It will wait for all tasks to complete and will not cancel scheduled tasks.

For example:

...
# create a thread pool
with ThreadPoolExecutor() as executor
	# submit tasks

This is functionally equivalent to the following manual creation and shutdown of the ThreadPoolExecutor:

...
# create a thread pool
executor = ThreadPoolExecutor()
# do things...
# shutdown a thread pool
executor.shutdown()

Which is also the same as the following with the default arguments to the shutdown() function stated explicitly:

...
# create a thread pool
executor = ThreadPoolExecutor()
# do things...
# shutdown a thread pool
executor.shutdown(wait=True, cancel_futures=False)

The context manager is so convenient that, like opening socket connections and files, using the context manager with the ThreadPoolExecutor is a best practice and should be considered a default usage pattern.

Now that we know how to automatically shut down a ThreadPoolExecutor using the context manager, let's look at a worked example.

Example of Using the ThreadPoolExecutor Context Manager

Let's develop a worked example using the ThreadPoolExecutor context manager.

First, let's define a task that will sleep for a moment to simulate an blocking IO operation.

The task() target task function takes a unique name, calls the sleep() function for a fraction of a second, then returns the unique name provided as an argument.

# mock test that works for moment
def task(name):
    sleep(0.5)
    return name

Next, we can create the thread pool using the context manager.

We will configure the ThreadPoolExecutor to have ten worker threads to match the ten tasks we intend to execute asynchronously.

# start the thread pool
with ThreadPoolExecutor(10) as executor:
	# ...

Next, we can submit our tasks to the thread pool for execution.

We will use the map() function, which will apply our task() function for each value in an iterable, in this case, a range of integers from 0 to 9.

We will iterate the results in the order that the tasks are submitted in a for loop and report the results with a print statement.

...
# submit tasks and process results
for result in executor.map(task, range(10)):
    # report the result
    print(f'Task done: {result}')

The thread pool will then be shut down automatically for us by the context manager once we are finished with it and leave the with block.

Tying this together, the complete example of using the ThreadPoolExecutor context manager is listed below.

# SuperFastPython.com
# example of using the thread pool with a context manager
from time import sleep
from concurrent.futures import ThreadPoolExecutor

# mock test that works for moment
def task(name):
    sleep(0.5)
    return name

# start the thread pool
with ThreadPoolExecutor(10) as executor:
    # submit tasks and process results
    for result in executor.map(task, range(10)):
        # report the result
        print(f'Task done: {result}')
    # thread pool is shutdown automatically
# all done, continue on
print('Done!')

Running the example first creates the thread pool using the context manager with ten worker threads.

We then submit tasks into the thread pool by calling the map() function and processing the results in the order that the tasks were submitted as the tasks are completed.

The result of each task is then reported with a print statement.

Once all tasks are completed, the thread pool is closed by an automatic call to the shutdown() function by the ThreadPoolExecutor context manager.

The program is then free to continue.
